Basic Questions:
1. What is Agentic AI?
Agentic AI refers to AI systems that can autonomously plan,
decide, and execute tasks to achieve a goal using reasoning, memory, and tools.

2. How is Agentic AI different from a chatbot?
A chatbot mainly responds to prompts. Agentic AI can break goals into steps, use tools/APIs, make decisions, and complete tasks autonomously.

3. What are the core components of an Agentic AI system?
LLM / reasoning model

4. What is an AI Agent?
An AI Agent is a software entity that perceives input, reasons about goals, 
and takes actions to achieve outcomes.

🏆 Real-World Design Questions

11. How would you build a customer support Agentic AI?
Workflow:
Input Ticket (Receives customer issue/request message from email, chat, or portal)
↓
Intent Agent (Identifies the purpose such as complaint, refund, support, or inquiry)
↓
Priority Agent (Determines urgency level like low, medium, or high priority)
↓
Knowledge Search Agent (Finds relevant solutions from FAQs, documents, or knowledge base)
↓
Reply Agent (Generates a clear and professional response for the customer)
↓
Escalation if needed (Transfers unresolved or critical cases to a human support team)

How projects works:

Read Email (Extracts and understands the customer email content)
↓
Detect Intent (Identifies request type such as complaint, refund, meeting, or inquiry)
↓
Choose Tone (Selects suitable response style like professional, empathetic, or formal)
↓
Generate Reply (Creates a contextual and professional email response)
↓
Send or Save Draft (Automatically sends the reply or stores it for human review)


13. How do you prevent hallucinations?
RAG with trusted sources: (Uses verified documents or databases 
so the model answers from real information instead of guessing)

Tool validation: (Checks results from APIs, calculators, 
or databases before using them in the final answer)

Human approval: (Allows a person to review sensitive 
or important responses before they are sent)

Guardrails (Applies rules and restrictions to block unsafe, false, 
or out-of-scope outputs)

Output checks (Runs final responses through validation tests for accuracy, format, and consistency)

🏆 Technical Questions

14. What is ReAct in Agentic AI?
ReAct = Reason + Act. The model reasons step-by-step and chooses actions/tools iteratively.

15. What is Tool Calling?
The model selects and invokes external functions/APIs to complete tasks.

16. What is RAG and how does it help agents?
RAG (Retrieval-Augmented Generation) retrieves external knowledge 
before generating answers, improving accuracy.

🏆 Production Questions

18. Challenges in deploying Agentic AI?
Cost control: (High API usage, repeated agent calls, 
and large model requests can increase operational expenses)

Latency: (Multiple reasoning steps, tool calls, and slow APIs can delay response time)

Hallucinations: (The model may generate incorrect or fabricated information with high confidence)

Security (Sensitive data may be exposed without proper access control, encryption, or secret management)

Prompt injection (Malicious user input can manipulate the agent into ignoring rules or leaking data)

Monitoring (Without logs and metrics, failures, misuse, and performance issues are hard to detect)

Reliability (Agents may fail due to API outages, tool errors, inconsistent outputs, or workflow breakdowns)

To recover :
Cost control (Use rate limits, caching, smaller models, token budgets, and usage alerts to reduce expenses)
Latency (Use faster models, parallel agent tasks, response streaming, caching, and optimized prompts)
Hallucinations (Use RAG with trusted data, fact-checking tools, output validation, and human review for critical tasks)
Security (Apply encryption, RBAC access control, secure secret storage, data masking, and audit logs)
Prompt injection (Sanitize inputs, isolate tools, enforce system rules, validate retrieved content, and restrict permissions)
Monitoring (Track logs, latency, token usage, errors, success rate, and create automated alerts/dashboards)
Reliability (Add retries, fallbacks, timeout handling, health checks, version control, and disaster recovery plans)

🏆 Frequently Asked Coding Questions
21. How do you create an agent loop? 
(Use a workflow loop where the agent thinks, takes action, observes results,
 and repeats until the goal is completed or stop condition is met)

22. How do you maintain state between agent calls?
 (Store conversation context, task progress, variables, and previous
  outputs in memory, database, or workflow state objects)

23. How do you parallelize multiple agents? 
(Run independent agents simultaneously using async tasks,
 threads, or workflow branches, then merge their outputs)

24. How do you retry failed tool calls? 
(Use retry logic with timeouts, exponential backoff, fallback tools,
 and error handling when APIs or tools fail)

25. How do you store memory vectors? 
(Convert text into embeddings and store them in vector databases 
for semantic search and retrieval)
